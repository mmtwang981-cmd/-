## Folder: `docs`

This folder contains product and agent-facing documentation for Morning Love Mail.

Current files:
- `prd.md` – product requirements and high-level intent for the Morning Love Mail MVP.
  - Describes the product goal, personas, content constraints, and MVP scope.
  - Also currently mirrors the core Agent rules to keep product + engineering context aligned.

Usage:
- When working on new features or refactors, read `docs/prd.md` first to align with:
  - product narrative,
  - persona details,
  - content quality constraints,
  - MVP boundaries (single-user, no auth, no real DB for now).

Future additions:
- More detailed UX flows or copy guidelines, if needed.
